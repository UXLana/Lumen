# Design Brief Generator

## When to Use

- Starting a new feature or project
- Documenting requirements before design work
- Creating specifications for handoff
- Aligning stakeholders on scope and goals

## Brief Structure

### 1. Project Overview
- Project name and description
- Goals and objectives
- Success metrics

### 2. User Context
- Target users/personas
- User needs and pain points
- User journey context

### 3. Requirements
- Functional requirements
- Non-functional requirements
- Constraints and limitations

### 4. Design Specifications
- Visual direction
- Interaction requirements
- Responsive behavior
- Accessibility requirements

### 5. Technical Considerations
- Platform/device targets
- Performance requirements
- Integration points

### 6. Timeline and Milestones
- Key deliverables
- Review checkpoints
- Dependencies

## Output Format

```markdown
# Design Brief: [Project Name]

## Overview
[Description and context]

## Goals
- Primary: [Main objective]
- Secondary: [Supporting objectives]

## Users
- Persona: [Target user]
- Context: [When/where they'll use this]

## Requirements
### Must Have
- [ ] Requirement 1
- [ ] Requirement 2

### Nice to Have
- [ ] Requirement 3

## Design Direction
[Visual and interaction guidance]

## Success Metrics
- [ ] Metric 1
- [ ] Metric 2
```

## User Input Required

$ARGUMENTS

---

Please provide information about the project:
- What feature/page/component needs a brief?
- Who are the target users?
- What are the main goals?
- Any known constraints?
