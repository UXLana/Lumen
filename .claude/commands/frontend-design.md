# Frontend Design

## When to Use

- Before implementing UI/UX changes (consult first)
- Establishing interaction patterns for new components
- Making design decisions about layouts and flows
- Reviewing accessibility considerations
- Exploring visual design alternatives
- Getting recommendations for micro-interactions

## Capabilities

### UX Patterns
- Navigation patterns (tabs, sidebars, breadcrumbs)
- Form design and validation feedback
- Loading states and skeleton screens
- Empty states and error handling
- Modal and dialog patterns
- Toast and notification patterns

### Interaction Design
- Hover and focus states
- Click/tap feedback
- Transition timing and easing
- Micro-animations
- Gesture support (swipe, drag)

### Visual Design
- Typography hierarchy
- Color usage and contrast
- Spacing and rhythm
- Visual weight and balance
- Icon and illustration usage

### Accessibility
- WCAG compliance guidance
- Keyboard navigation patterns
- Screen reader considerations
- Color contrast requirements
- Focus management
- Reduced motion support

## Consultation Format

When consulted, I provide:

1. **Analysis** of the current state/problem
2. **Options** with pros/cons for each approach
3. **Recommendation** with rationale
4. **Implementation notes** for the chosen approach
5. **Accessibility considerations**

## Integration

Other skills should invoke `/frontend-design` before:
- Making layout changes
- Adding new interaction patterns
- Modifying visual hierarchy
- Implementing animations
